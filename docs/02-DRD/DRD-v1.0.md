## 2. 设计原则

### 2.1 设计愿景

打造一款具有现代游戏风格的网页贪吃蛇，界面简洁明快，操作直观流畅。游戏风格偏向「赛博霓虹」，通过鲜明的色彩对比和流畅的动画效果，营造沉浸式的游戏体验。排行榜页面强调竞技氛围，通过实时更新和推送通知增强竞争感。

### 2.2 设计原则

1. **游戏优先** - 游戏画布是核心，UI 不喧宾夺主
2. **即时反馈** - 每次操作都有视觉或音效反馈
3. **信息清晰** - 得分、排名等关键信息一目了然
4. **轻量联网** - 网络请求不阻塞游戏运行
5. **跨设备一致** - 在桌面和平板上保持体验一致

## 5. 组件规范

### 5.1 游戏画布组件 (GameCanvas)

**职责**: 使用 Canvas 2D API 渲染游戏世界

```typescript
interface GameCanvasProps {
  width: number;           // 画布宽度 (像素)，默认 600
  height: number;          // 画布高度 (像素)，默认 600
  gridSize: number;        // 格子大小 (像素，默认 15)
  snake: SnakeState;       // 当前蛇的状态
  foods: FoodState[];      // 所有食物的位置
  gameStatus: GameStatus;  // 游戏状态
}

interface SnakeState {
  segments: Position[];    // 蛇身坐标数组
  direction: Direction;
  score: number;
  length: number;
  skin?: string;           // 皮肤 ID（P2）
}

interface FoodState {
  position: Position;
  type: 'normal';          // v1.0 仅普通食物
}

interface Position {
  x: number;
  y: number;
}

type Direction = 'up' | 'down' | 'left' | 'right';
type GameStatus = 'idle' | 'running' | 'paused' | 'gameover';
```

**渲染要求**:
- 使用 `requestAnimationFrame` 进行流畅渲染
- 蛇身渲染：每个格子 15x15px，带霓虹发光效果
- 食物渲染：带闪烁动画（透明度 0.6 ~ 1.0 循环）
- 背景：深色网格（#161B22 底色 + #21262D 网格线）

**性能要求**:
- 渲染帧率 >= 30 FPS
- 避免在渲染循环中创建新对象（对象复用）

## 6. 响应式设计

### 6.1 断点定义

| 断点 | 宽度 | 设备 |
|------|------|------|
| LG | >= 1200px | 桌面（完整布局） |
| MD | >= 768px | 平板（侧栏折叠） |
| SM | < 768px | 手机（简化浏览） |

### 6.2 各断点适配

#### 桌面 (LG, >= 1200px)

- 首页：三栏布局（Logo + 开始游戏 + 排行榜/统计）
- 游戏页：画布 600x600px + 右侧面板
- 排行榜：完整表格布局

#### 平板 (MD, 768-1199px)

- 首页：排行榜/统计折叠到开始游戏按钮下方
- 游戏页：画布缩放至 450x450px，右侧面板折叠
- 排行榜：表格列数减少（隐藏总场次）

```
┌────────────────────────┐
│ [返回]    贪吃蛇  得分  │
├────────────────────────┤
│                        │
│   Canvas 450x450       │
│                        │
│   最高分: 1250          │
│   [暂停] [退出]         │
│                        │
├────────────────────────┤
│ 长度: 15    速度: 120ms │
└────────────────────────┘
```

#### 手机 (SM, < 768px)

- 游戏页：画布缩放至 100% 宽度，保持正方形比例
- 排行榜：单列卡片布局（非表格）
- 所有功能可用，但体验降级为触屏操作（虚拟方向键）

### 6.3 画布自适应

```typescript
// 根据可用空间计算画布缩放比例
const getCanvasScale = (containerWidth: number, containerHeight: number) => {
  const baseSize = 600;
  const padding = 32;
  const availableWidth = containerWidth - padding;
  const availableHeight = containerHeight - padding;
  return Math.min(
    availableWidth / baseSize,
    availableHeight / baseSize,
    1.0  // 最大不放大
  );
};
```

## 8. 动画规范

### 8.1 动画原则

1. **服务于游戏** - 动画用于增强游戏反馈，不干扰 gameplay
2. **性能优先** - 游戏画布动画使用 Canvas 渲染，UI 动画使用 CSS transform
3. **时长适中** - UI 动画 200-300ms，游戏内动画 100-200ms
4. **可关闭** - 提供「减少动画」选项

### 8.2 动画列表

| 动画 | 时长 | 缓动 | 触发条件 |
|------|------|------|---------|
| 蛇吃到食物 | 150ms | ease-out | 蛇头接触食物 |
| 蛇死亡淡出 | 300ms | ease-in | 撞墙/自身 |
| 食物闪烁 | 循环 | sine | 持续 |
| 得分数字滚动 | 200ms | ease-out | 得分变化 |
| 按钮点击 | 150ms | ease-out | 点击 |
| 页面切换 | 300ms | ease-in-out | 路由切换 |
| 排名更新 | 400ms | cubic-bezier | 排行榜刷新 |
| 推送通知进入 | 300ms | ease-out | 新通知 |

### 8.3 死亡淡出效果

```typescript
// 蛇死亡时的淡出效果
function renderDeathFade(alpha: number) {
  // alpha 从 1.0 渐变到 0.3，持续 300ms
  ctx.globalAlpha = alpha;
  // 渲染蛇身
  ctx.globalAlpha = 1.0;
```
